## 一、产品概述

一个**统一品牌的跨境电商 AI 工具集合站**，覆盖独立站卖家从选品到投放、从内容到售后的日常运营场景。每个工具解决一个具体痛点，深度对接 Google Ads、GMC、GA4、Amazon 等平台 API。

**一句话定位**：跨境电商的 AI 瑞士军刀 —— 专业、深度、即开即用。

**产品形态**：Web SaaS 工具站（未来可扩展为 Shopify App）

### 2.2 工具详解

#### ① 选品发现器

| 维度 | 说明 |
|------|------|
| 解决问题 | "想做独立站，不知道卖什么" |
| 数据来源 | Amazon Ads API + Seller API（多区域：北美/欧洲/远东） |
| 核心能力 | AI 分析热门品类、趋势变化、竞争强度、利润空间 |
| 输出 | 选品推荐报告（含市场规模、竞品数量、建议售价） |

#### ② 品类调研器

| 维度 | 说明 |
|------|------|
| 解决问题 | "这个品类值不值得做？" |
| 数据来源 | Google Keyword Planner API + Google Trends |
| 核心能力 | 近 30 天搜索量、搜索趋势、竞争度、CPC 参考、季节性分析 |
| 输出 | 品类调研报告（含市场趋势图、关键词列表、竞争度评分） |

#### ③ Google 广告生成器

| 维度 | 说明 |
|------|------|
| 解决问题 | "我不会写广告文案，也不会操作 Google Ads" |
| 数据来源 | Google Ads API + Google Keyword Planner |
| 核心能力 | 输入行业+产品 → 自动获取关键词 → AI 生成广告标题+描述 → **直接创建广告系列** |
| 支持类型 | 搜索广告、购物广告、效果最大化广告（PMax） |
| 对接深度 | OAuth 授权 → 读取账户数据 → 一键发布广告 |

#### ④ 申诉信生成器

| 维度 | 说明 |
|------|------|
| 解决问题 | "账号被封了，不知道怎么申诉" |
| 模板类型 | GMC 封号、Google Ads 拒登、广告账户被封、政策违规等 |
| 核心能力 | 选择封号类型 → 输入店铺信息 → AI 基于成功案例模板生成申诉信 |
| 差异化 | **基于真实成功案例打磨的模板库**（需要行业经验沉淀） |

#### ⑤ Feeds 优化器

| 维度 | 说明 |
|------|------|
| 解决问题 | "我的 Google Shopping 效果差" |
| 数据来源 | Google Content API + Merchant Center |
| 核心能力 | 读取 GMC 产品 Feed → AI 分析标题/描述质量 → 自动优化 → 回写 GMC |
| 优化维度 | 标题优化、描述补充、分类纠正、属性完善、关键词注入 |
| 对接深度 | 需要 Google Ads Manager + GMC 账户权限 |

#### ⑥ AI 博客生成器

| 维度 | 说明 |
|------|------|
| 解决问题 | "独立站需要内容但没人写" |
| 数据来源 | Amazon ASIN（产品信息）+ 关键词输入 |
| 核心能力 | 基于产品信息自动生成 SEO 友好的营销博客 |
| 发布 | 支持一键发布到 Shopify 店铺 |
| 内容类型 | 产品评测、使用指南、对比文章、行业趋势 |

#### ⑦ 促销活动生成器

| 维度 | 说明 |
|------|------|
| 解决问题 | "大促不知道怎么策划" |
| 核心能力 | 输入活动类型 → AI 分析竞品促销策略 → 生成完整促销方案 |
| 输出 | 促销策略 + 定价建议 + 中英文文案 + PDF/PPT 报告 |
| 活动类型 | 黑五、圣诞、Prime Day、清仓、新品上市等 |

#### ⑧ 受众解决方案

| 维度 | 说明 |
|------|------|
| 解决问题 | "广告投给谁效果好？" |
| 数据来源 | Google Ads API |
| 核心能力 | 分析产品线+目标市场 → AI 生成精准受众包 → **直接推送到 Google Ads** |
| 差异化 | 支持产品线优先级设置、自动受众包优化 |

#### ⑨ 数据周报

| 维度 | 说明 |
|------|------|
| 解决问题 | "每周看数据太累了" |
| 数据来源 | GA4（OAuth 正式授权，非 Chrome 登录态） |
| 核心能力 | 自动汇总周数据 → AI 生成分析洞察 → 推送报告 |
| 输出 | 流量、转化、收入趋势 + AI 异常分析 + 改进建议 |

#### ⑩ 社媒自动投放

| 维度 | 说明 |
|------|------|
| 解决问题 | "社交媒体要发内容但没精力" |
| 支持平台 | Facebook、Instagram、Twitter、YouTube、LinkedIn |
| 核心能力 | 产品图片 → AI 生成多类型营销内容 → 自动发布 |
| 差异化 | **自研内容生成引擎**（不依赖第三方） |

## 三、上线优先级

按「需求刚性 × 竞品差异化空间 × 开发可行性」排序：

| 批次 | 工具 | 核心理由 |
|------|------|---------|
| **第一批** | ③ 广告生成器 | 最高频刚需，对接 Google Ads API 后壁垒高 |
| **第一批** | ④ 申诉信生成器 | 痛点极强（封号=停业），行业经验壁垒最大 |
| **第一批** | ⑤ Feeds 优化器 | 竞品弱项，差异化空间大 |
| **第二批** | ⑥ 博客生成器 | 需求稳定，SEO 自身也能引流 |
| **第二批** | ⑧ 受众解决方案 | 对接 Google Ads，与广告生成器形成组合 |
| **第二批** | ⑨ 数据周报 | 用户粘性工具（每周必用） |
| **第三批** | ② 品类调研器 | 需要 Google API 数据积累 |
| **第三批** | ① 选品发现器 | 需要 Amazon API 对接 |
| **第三批** | ⑦ 促销活动生成器 | 可配合大促节点上线 |
| **第三批** | ⑩ 社媒自动投放 | 自研引擎需要时间打磨 |

## 四、商业模式

- **免费增值（Freemium）**：每个工具提供免费额度，付费解锁无限使用和高级功能
- **团队/Agency 版**：多账号、API 接口、白标、批量操作

## 五、与方案 二/三 的关系

方案二/B/C 不是三选一，是**一个产品的三层深度**：

```
第一层（工具）→ 方案一
  免费/低价工具引流，建立用户基础
  申诉信、博客、选品、品类调研

第二层（广告）→ 方案二（AdPilot）
  C 中的广告生成器、Feeds优化器、受众方案的深度版
  从"生成"到"投放→追踪→优化"闭环

第三层（决策）→ 方案三（OpsGenie）
  全链路数据 + AI 运营建议
  所有工具数据汇聚成决策中台
```

**方案一是免费引流层，方案二是付费核心，方案三是高阶增值。**

### 功能归属对照

| 方案一的工具 | → 升级为方案二模块 | → 汇入方案三数据 |
|---------|-----------------|----------------|
| 广告生成器 | 素材工坊 + 智能投手 | 广告效果数据 |
| Feeds 优化器 | 效果中心 | 产品表现数据 |
| 受众解决方案 | 智能投手（受众推荐） | 受众转化数据 |
| 数据周报 | — | 健康体检数据源 |
| 品类调研 | — | 趋势预测数据源 |

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 竞品已有先发优势 | 中 | 高 | 聚焦差异化（行业经验+AI质量+中国卖家场景） |
| Google API 政策变更 | 中 | 高 | 多平台分散风险；及时跟进 API 更新 |
| 工具之间缺乏粘性 | 中 | 中 | 数据打通、组合推荐、账户体系统一 |
| 免费用户不转化 | 中 | 中 | 精细化 Free/Pro 切分；企业级功能驱动付费 |
| AI 生成质量参差 | 中 | 中 | 行业规则库 + 人工审核模板；持续收集反馈迭代 |
